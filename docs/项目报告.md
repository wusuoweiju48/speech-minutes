# 基于语音识别的课堂会议纪要自动生成工具

## 一、项目背景与目标

### 1.1 项目背景

在高校课堂教学中，学生需要记录大量的课堂笔记和会议内容。传统的人工记录方式效率低下，容易遗漏关键信息，且无法快速生成结构化的总结。随着语音识别技术和大语言模型的快速发展，利用技术手段实现课堂内容的自动记录和智能总结成为可能。

### 1.2 项目目标

本项目旨在开发一个基于语音识别的课堂会议纪要自动生成工具，实现以下功能：

1. **实时语音识别**：通过麦克风采集课堂音频，实时转换为文字
2. **发言人标记**：支持手动标记不同发言人，便于区分师生对话
3. **智能纪要生成**：利用大语言模型自动提取摘要、关键信息和待办事项
4. **多格式导出**：支持 Markdown、Word、纯文本三种格式导出

## 二、系统总体设计

### 2.1 系统架构

本系统采用"轻前端 + 轻后端 + 开源模型 + LLM"的分层架构：

```
┌─────────────────────────────────────────┐
│  前端 (HTML + JavaScript)                │
│  - 录音与切片 (Web Audio API)            │
│  - WebSocket 客户端                      │
│  - 发言人标记 UI                         │
│  - 实时转写展示                          │
│  - 纪要渲染与导出                        │
└──────────────┬──────────────────────────┘
               │ WebSocket / HTTP
┌──────────────▼──────────────────────────┐
│  后端 (Python FastAPI)                   │
│  - WebSocket 接收音频流                  │
│  - openai-whisper 本地推理               │
│  - GLM API 调用 (纪要生成)               │
└─────────────────────────────────────────┘
```

### 2.2 技术选型

| 层级 | 技术 | 选型理由 |
|---|---|---|
| 前端 | 纯 HTML + JavaScript | 零构建工具，双击即可运行，降低部署复杂度 |
| 后端 | Python FastAPI | 轻量级异步框架，原生支持 WebSocket |
| 实时 ASR | openai-whisper | OpenAI 开源模型，中文识别效果好，本地部署保护隐私 |
| 纪要生成 | 智谱 GLM-4.7-Flash | 国产大语言模型，中文理解能力强，API 调用成本低 |
| 音频采集 | Web Audio API | 浏览器原生 API，支持 16kHz PCM 采样，兼容性好 |

### 2.3 数据流设计

```
麦克风 → Web Audio API (16kHz PCM) → WebSocket → 后端缓冲 (3秒切片)
    → openai-whisper 转写 → 返回前端显示 → 用户点击"生成纪要"
    → GLM API 提取结构化信息 → 前端渲染纪要卡片 → 导出文件
```

## 三、语音处理算法原理

### 3.1 音频预处理

#### 3.1.1 采样与量化

- **采样率**：16kHz（满足语音识别的奈奎斯特要求）
- **量化位数**：16bit PCM（动态范围约 96dB）
- **声道数**：单声道（课堂场景无需立体声）

#### 3.1.2 音频分帧

采用固定 3 秒切片策略，每 48000 个采样点（16000 × 3）为一帧发送至后端。相比 VAD（语音活动检测）方案，固定切片实现简单，延迟可控，适合课堂场景。

#### 3.1.3 静音检测

在送入 Whisper 模型前，计算音频帧的 RMS（均方根）能量：

$$RMS = \sqrt{\frac{1}{N}\sum_{i=1}^{N}x_i^2}$$

当 RMS 低于阈值（默认 500）时，判定为静音，跳过转写以节省计算资源。

### 3.2 Whisper 模型原理

#### 3.2.1 模型架构

Whisper 采用 Encoder-Decoder Transformer 架构：

- **编码器（Encoder）**：将音频的 Mel 频谱图编码为高维特征向量
  - 输入：80 维 Mel 频谱图（25ms 窗口，10ms 帧移）
  - 位置编码：正弦位置编码
  - 注意力机制：多头自注意力（Multi-Head Self-Attention）

- **解码器（Decoder）**：基于编码器输出和已生成的 token，自回归生成文本
  - 交叉注意力（Cross-Attention）：关注编码器输出的相关区域
  - 掩码自注意力（Masked Self-Attention）：防止未来信息泄露

#### 3.2.2 多任务预训练

Whisper 通过多任务预训练实现语音识别、翻译、语言识别等能力：

- `<|transcribe|>`：语音转文字
- `<|translate|>`：语音翻译为英文
- `<|language:zh|>`：指定中文识别

#### 3.2.3 幻觉过滤

Whisper 模型在处理静音或噪音时可能产生幻觉输出（如"字幕by索兰娅"、"谢谢观看"等）。本系统实现基于规则的幻觉过滤：

1. 维护常见幻觉模式列表
2. 检测重复字符模式（如"谢谢谢谢谢谢..."）
3. 匹配到幻觉模式时丢弃结果

### 3.3 LLM 纪要生成

#### 3.3.1 Prompt 工程

采用结构化 Prompt 引导 GLM 模型生成规范的会议纪要：

```
你是一个课堂会议纪要助手。请根据以下课堂转写记录，生成结构化的会议纪要。
要求：
1. 摘要：用一两句话概括本次课堂的核心内容
2. 关键信息：列出课堂中提到的 3-5 个重点知识点
3. 待办事项：列出提到的作业、任务或后续安排
请严格以 JSON 格式返回。
```

#### 3.3.2 结果解析

后端接收 GLM 返回的 JSON 数据，校验必要字段（summary、key_points、action_items）后返回前端渲染。若 LLM 调用失败，自动降级为模板摘要。

const source = audioCtx.createMediaStreamSource(stream);
const processor = audioCtx.createScriptProcessor(4096, 1, 1);

processor.onaudioprocess = (e) => {
  const float32 = e.inputBuffer.getChannelData(0);
  // float32 转 int16
  const int16 = new Int16Array(float32.length);
  for (let i = 0; i < float32.length; i++) {
    const s = Math.max(-1, Math.min(1, float32[i]));
    int16[i] = s < 0 ? s * 0x8000 : s * 0x7FFF;

### 4.2 后端实现

#### 4.2.1 FastAPI 服务

```python
@app.websocket("/ws/transcribe")
async def ws_transcribe(ws: WebSocket):
    await ws.accept()
    buf = bytearray()
    CHUNK = 16000 * 2 * 3  # 3秒 16kHz 16bit mono

    while True:
        msg = await ws.receive()
        if "bytes" in msg:
            buf.extend(msg["bytes"])
            while len(buf) >= CHUNK:
                chunk = bytes(buf[:CHUNK])
                buf = buf[CHUNK:]
                text = transcribe_audio(chunk)
                if text:
                    await ws.send_json({"type": "transcript", "text": text})
```

### 4.3 错误处理

| 异常场景 | 处理策略 |
|---|---|
| 麦克风权限被拒 | 前端 toast 提示，自动切换仿真模式 |
| WebSocket 连接失败 | 顶部红色横幅提示，降级为仿真模式 |
| Whisper 模型加载失败 | 后端日志警告，自动使用仿真数据 |
| GLM API 调用失败 | 降级为模板摘要，保证系统可用 |
| 音频为静音 | 静音检测跳过，节省计算资源 |
| Whisper 幻觉输出 | 规则过滤，丢弃无效结果 |

## 六、总结与展望

### 6.1 项目总结

本项目成功实现了一个基于语音识别的课堂会议纪要自动生成工具，主要成果包括：

1. **完整的系统架构**：前端纯 HTML + 后端 FastAPI + WebSocket 实时通信，部署简单
2. **本地化 ASR**：使用 openai-whisper 本地推理，保护隐私，无需付费
3. **智能纪要生成**：集成智谱 GLM 大模型，自动生成结构化会议纪要
4. **鲁棒的错误处理**：静音检测、幻觉过滤、仿真降级，保证系统稳定运行
5. **良好的用户体验**：Anthropic 暖色调 UI 设计，操作直观，支持多格式导出

### 6.2 不足与改进方向

1. **说话人分离**：当前采用手动标记，未来可集成 pyannote.audio 实现自动说话人分离
2. **实时性优化**：可探索流式 Whisper（Whisper Streaming）降低转写延迟
3. **离线 LLM**：当前依赖在线 GLM API，未来可部署本地大模型（如 Qwen、ChatGLM）
4. **多语言支持**：Whisper 本身支持多语言，可扩展支持英语、日语等课堂

### 6.3 应用前景

本系统可广泛应用于高校课堂、学术讲座、企业会议等场景，帮助用户快速获取结构化的会议纪要，提升信息记录和知识管理效率。
