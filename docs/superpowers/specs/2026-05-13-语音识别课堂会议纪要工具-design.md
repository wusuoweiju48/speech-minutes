# 基于语音识别的课堂会议纪要自动生成工具 — 设计文档

## 1. 项目概述

**课程**：数字语音处理  
**题目**：基于语音识别的课堂会议纪要自动生成工具  
**形态**：Web 页面（浏览器访问）  
**核心能力**：实时语音识别、发言人手动标记、智能纪要自动生成（摘要 / 关键信息 / 待办事项 / 话题分段）。

## 2. 设计目标与约束

- **目标**：在有限时间内完成一个可运行、可演示、可写进报告的完整系统。
- **约束**：
  - 调用开源/第三方 ASR，不自研声学模型。
  - 单页应用，无用户系统、无数据库。
  - 本地运行，答辩时一键启动。
  - 5 分钟演示视频需覆盖完整使用链路。

## 3. 架构设计

采用"轻前端 + 轻后端 + 开源模型 + LLM 兜底"的分层架构：

```
┌─────────────────────────────────────────┐
│  前端 (HTML + JS)                        │
│  - 录音与切片 (Web Audio API)            │
│  - WebSocket 客户端                      │
│  - 发言人标记 UI                         │
│  - 实时转写展示                          │
│  - 纪要渲染与导出                        │
└──────────────┬──────────────────────────┘
               │ WebSocket / HTTP
┌──────────────▼──────────────────────────┐
│  后端 (Python FastAPI)                   │
│  - WebSocket 接收音频流                  │
│  - faster-whisper 本地推理               │
│  - LLM API 调用 (纪要生成)               │
└─────────────────────────────────────────┘
```

## 4. 技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| 前端 | 纯 HTML + JavaScript | 零构建工具，双击即可打开 |
| 后端 | Python FastAPI | 轻量，原生支持 WebSocket |
| 实时 ASR | faster-whisper (本地) | 开源，中文效果够用，速度优于原版 Whisper |
| 说话人区分 | 前端手动标记 + 可选 pyannote.audio 离线分析 | 手动标记保证演示可控；离线分析作为加分项 |
| 纪要生成 | LLM API (DeepSeek / 通义千问 / 文心一言) | Prompt 工程驱动，返回结构化 JSON |
| 存储 | 纯前端内存 | 刷新页面即清空，支持导出 Markdown / TXT |

## 5. 功能模块详述

### 5.1 前端录音与分段
- 使用 `MediaRecorder` 或 `Web Audio API` 采集麦克风。
- **固定 3 秒切片**（不实现复杂 VAD），通过 WebSocket 发送至后端。
- 维护 `currentSpeaker` 状态，默认 `"老师"`。

### 5.2 实时 ASR 服务
- FastAPI 提供 WebSocket 端点 `/ws/transcribe`。
- 收到音频片段后临时保存为 WAV，调用 `faster-whisper` 识别。
- 识别结果即时推回前端。
- **边界处理**：若返回文字为空则静默丢弃；前端按标点或时间顺序做简单拼接。

### 5.3 发言人文本组装
- 前端收到识别文本后，前缀拼接当前选中的发言人标签，例如 `【老师】今天我们来复习第三章。`
- 句子存入数组，实时渲染到页面"会议记录"区域。

### 5.4 智能纪要生成
- 录音结束后，前端将完整文本通过 POST `/generate_summary` 发送至后端。
- 后端将文本嵌入 Prompt，调用 LLM API，要求返回如下 JSON：
  ```json
  {
    "summary": "一句话摘要",
    "key_points": ["重点1", "重点2"],
    "action_items": ["小明负责整理实验报告（下周交）"],
    "segments": [{"topic": "复习第三章", "start": "...", "end": "..."}]
  }
  ```
- 前端解析 JSON，渲染为结构化纪要卡片。

### 5.5 导出与重置
- 导出 Markdown：将"原始记录 + 结构化纪要"拼接为 Markdown 文本，触发浏览器下载。
- 重置：清空前端所有状态，回到初始页面。

## 6. 数据流

```
麦克风
  ↓
[前端: 3秒切片] ──WebSocket──→ [后端: faster-whisper]
                                   ↓
用户点击发言人按钮 ←──文字流───────┘
  ↓
[前端: 打标签 → 显示记录]
  ↓
点击"生成纪要"
  ↓
[后端: Prompt + LLM API] → [前端: 显示纪要]
  ↓
[导出 Markdown / 复制全文]
```

## 7. 页面结构与交互

### 7.1 页面布局（单页，自上而下）

```
┌─────────────────────────────┐
│  标题：课堂会议纪要生成工具      │
├─────────────────────────────┤
│  [开始录音] [停止录音] [重置]   │
├─────────────────────────────┤
│  [老师] [学生A] [学生B] [+]   │  ← 高亮当前选中
├─────────────────────────────┤
│  【老师】今天我们来复习第三章。  │  ← 实时转写区（自动滚动）
│  【学生A】老师，第三章哪部分？   │
│         ...                 │
├─────────────────────────────┤
│  [一键生成会议纪要]            │  ← 停止录音后可用
├─────────────────────────────┤
│  📋 会议摘要：...             │  ← 纪要展示区
│  ⭐ 关键信息：...             │
│  ✅ 待办事项：...             │
│  [导出 Markdown] [复制全文]   │
└─────────────────────────────┘
```

### 7.2 操作流程
1. 打开页面，自动连接后端 WebSocket；失败则顶部红条提示。
2. 点击"开始录音"，请求麦克风权限，按钮变红。
3. 发言过程中，手动切换发言人按钮；文字实时滚入中间区域。
4. 点击"停止录音"，"一键生成会议纪要"按钮变为可用。
5. 点击生成，等待 2-3 秒后下方展开纪要。
6. 导出或复制结果。

### 7.3 错误处理（最小可行版）
- 麦克风权限被拒：`alert("请允许麦克风权限")`。
- 后端未启动：页面顶部红条提示"后端连接失败，请检查 127.0.0.1:8000"。
- ASR 某段无结果：静默丢弃，前端无感知。

### 8.3 常见答辩问题预案
- **为何不做实时自动说话人分离？**  
  → 课堂环境嘈杂，实时 diarization 准确率低；采用交互式标记保证准确性，并预留离线分析接口。
- **Whisper 是否自研？**  
  → 使用 OpenAI 开源预训练模型，本地部署 faster-whisper 推理，重点在于语音处理 pipeline 的系统集成。
- **延迟如何？**  
  → 采用 3 秒固定切片，在目标硬件上实现准实时转写，满足课堂纪要场景需求。

## 9. 可选扩展（时间充裕时）

- **离线说话人自动分离**：会议结束后，将完整录音传给 `pyannote.audio` 做说话人聚类，与手动标记结果对比。
- **话题自动分段**：基于 LLM 返回的 `segments` 在转写文本中插入段落标题。
- **多语言支持**：Whisper 本身支持多语言，切换模型参数即可。
